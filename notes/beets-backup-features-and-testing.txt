Beets Backup — Feature List, Known Limitations, and Critical Testing Areas
Last updated: 2026-04-21

================================================================================
SECTION 1: COMPLETE FEATURE LIST
================================================================================

FILE MANAGER — LAYOUT AND NAVIGATION
- Dual-pane file manager (single pane and split pane modes)
- Source/Destination paradigm for file transfers
- Drive tree sidebar with lazy-loading folder expansion
- Drive usage rings showing percentage used, free space, total space
- Back/Forward/Up navigation with full history (browser-style)
- Breadcrumb-style path display
- Drive refresh button (hot-plug USB support)
- Drive load errors caught per-drive (ejected drive during enumeration does not break list)

FILE LIST
- File list with columns: Name (with icon), Type, Size, Modified
- Column sorting (ascending/descending) with folders always first
- Multi-select file support (Ctrl+Click, Shift+Click)
- Context menus (right-click) with keyboard shortcuts shown
- Folder size calculation (async, with "Calculating..." indicator; UI thread-safe)
- Hidden file support (preserved during copy)

SEARCH
- Deep recursive search across entire directory trees
- Extension-based search (prefix with .)
- Live filter search on current folder
- Search status indicators (Searching..., No results found)
- Clear search to restore normal view

FILE OPERATIONS
- Copy files/folders between panes
- Move (cut) files/folders between panes
- Delete files/folders (sends to Recycle Bin)
- Rename files/folders via dialog
- Clipboard operations (Copy, Cut, Paste) in single pane mode
- Drag and drop between panes

TRANSFER OPTIONS
- Transfer mode selection: Skip Existing (Merge), Keep Both, Replace, Mirror (Sync)
- Skip Existing compares both file size AND last modified timestamp
- Keep Both auto-renames with -1, -2 suffixes
- Replace overwrites existing files
- Mirror copies new/changed files then deletes destination-only files (with confirmation dialog)
- Remove NTFS permissions option during transfer
- Verify checksums (SHA256) after copy option
- Pre-transfer free space check (skipped safely for UNC/network paths)
- Timestamp preservation: destination last-write time matches source after copy

TRANSFER PROGRESS
- Transfer progress bar with percentage and ETA
- Pause/Resume active transfers (fresh ManualResetEventSlim per transfer)
- Stop (cancel) active transfers
- Disk-full detection with specific error reporting
- Locked-file detection with specific error reporting; VSS fallback for locked files (requires elevation)
- Directory failure tracking (separate from file failures)
- Per-file error tracking: path + reason recorded for every failed file (capped at 200 per job)

BANDWIDTH THROTTLING
- "Limit Speed" checkbox in main toolbar (fixed 10 MB/s cap)
- Per-job speed picker in scheduler dialog (1–100 MB/s options)
- Chunked streaming with stopwatch-based rate limiting in TransferService

VOLUME SHADOW COPY (VSS)
- VssSnapshotService: creates on-demand VSS snapshots via COM P/Invoke to vssapi.dll
- Snapshot cache: one snapshot per volume per session (avoids duplicate VSS overhead)
- Translates normal file paths to shadow-copy device paths for copying locked files
- E.g. C:\Users\foo\bar.db → \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy3\Users\foo\bar.db
- Snapshots cleaned up on Dispose (DeleteSnapshots call removes shadow copies from disk)
- Requires elevation; throws InvalidOperationException with a helpful message if VSS fails
- Locked-file handling in TransferService has a VSS fallback path

FILE VERSIONING
- Optional per-job versioning: before overwriting a destination file, the existing copy is moved
  to a hidden .versions/ subfolder at the destination root
- Archive naming format: {name}__{yyyy-MM-dd_HH-mm-ss}{ext} (counter suffix for sub-second duplicates)
- Configurable retention: MaxVersions prunes the oldest archived copies beyond the limit
- .versions folder is marked Hidden so it doesn't appear in normal file browsing
- Versioning and compression are mutually exclusive (enforced in wizard UI and at job build time)
- "Previous Versions…" context menu item opens PreviousVersionsDialog for any file in a versioned destination
- Previous Versions dialog: lists archived versions newest-first, with Restore and Delete commands
- Restore is reversible: the current live copy is archived before being overwritten
- VersioningService.FindVersionsRoot() auto-discovers the destination root by walking up the directory tree

COMPRESSION
- Optional per-job compression: backs up sources as a single .zip archive at the destination
- Replaces direct file copy — the entire source tree is zipped into one archive file
- Mutually exclusive with file versioning

SCHEDULED BACKUP JOBS
- Scheduled backup jobs with custom names
- Multiple source folders per scheduled job
- Recurring backups: Daily, Weekly, Every 6 Hours, Every 12 Hours
- One-time scheduled backups
- Per-job transfer mode selection
- Per-job permission stripping option
- Per-job checksum verification option
- Per-job exclusion filters (by extension pattern and exact name)
- Per-job speed limit option
- Per-job file versioning option (MaxVersions configurable)
- Per-job compression option (mutually exclusive with versioning)
- Backup size estimation with "Estimate Size" button (respects exclusion filters)
- Scheduled Jobs dialog to view/manage/delete jobs
- Background scheduler with 1-minute resolution
- Thread-safe job scheduler: SnapshotJob() clones job data before background execution
- Scheduler errors surfaced in main window status bar via SchedulerError event
- Missed backup detection on startup with Run All Now / Skip All dialog
- LastRun tracking per job for accurate missed-backup detection

BACKUP WIZARD
- Multi-step Backup Wizard dialog with 7 steps and a stepper-dot progress indicator
- Step 1 – Backup Type: choose One-time now, Scheduled (one-time), or Recurring
- Step 2 – When (skipped for "One-time now"): date/time picker + recurrence interval selection
- Step 3 – Source: multi-folder picker or full-drive selection
- Step 4 – Destination: folder picker with same-source-drive detection warning
- Step 5 – Mode: transfer mode selection with plain-English descriptions of each mode
- Step 6 – Options: checksum verify, strip permissions, throttle, file versioning (MaxVersions),
  compression to .zip, and exclusion filters (all optional; all default off)
- Step 7 – Review: human-readable summary of all choices + async disk-space estimate
- "One-time now" path runs the job immediately; finish button label is "Start Backup Now"
- Scheduled/Recurring path adds the job to the scheduler; finish button label is "Schedule Backup"
- Pre-flight disk-space gate: if estimated size exceeds free space, a warning MessageBox fires
  before finalizing; user can cancel or proceed
- Wizard validates source and destination before allowing Next navigation

WINDOWS TASK SCHEDULER INTEGRATION
- WindowsTaskSchedulerService: registers Beet's Backup jobs with Windows Task Scheduler via schtasks.exe
- Registered tasks fire even when the app is closed or after a reboot (--run-job {id} argument)
- Task naming convention: BeetsBackup_{jobId} (GUID, no dashes) — easy to identify in Task Scheduler
- Runs as the interactive user (LIMITED privilege level — no elevation required)
- Supports all four recurrences: ONCE, DAILY, WEEKLY, HOURLY /MO N
- ReconcileAll() on startup re-registers any tasks removed from Task Scheduler
- Unregister() deletes the Windows task when a job is deleted in Beet's Backup

TOAST NOTIFICATIONS
- ToastNotifier: Windows balloon/toast notifications via the system tray NotifyIcon
- Three severity levels: Success (Info icon), Warning (Warning icon), Error (Error icon)
- Fires on backup job completion: success, partial errors, or outright failure/cancellation
- Uses WinForms NotifyIcon.ShowBalloonTip — renders as native Win10/11 toast; no AppUserModelId
  or manifest requirements

BACKUP LOG
- Backup log with detailed history (timestamp, job, status, source, stats, details)
- Log statuses: Scheduled, Running, Complete, Failed, Complete with N error(s)
- Running jobs show progress bar in log
- Per-file error records on log entries (FilesFailed count + List<FileError>)
- "View Errors" button enabled when selected entry has file errors
- Export log to CSV (newline-safe escaping)
- "Open Log Folder" button opens %LocalAppData%\Beet's Backup\ in Explorer
- Clear log (persists to disk)
- Log capped at 500 entries (oldest dropped)
- Stale "Running" entries auto-marked as Failed on startup
- Debounced save (1-second delay) prevents I/O on every status tick

DATA VISUALIZATION
- Visual Mode toolbar button toggles between List view and donut pie chart
- Pie chart shows top 10 largest items in the current folder by size
- "Other" slice (muted gray) accounts for all items beyond top 10
- 10 distinct colors per slice (blue, rose, green, amber, violet, red, sky, orange, emerald, fuchsia)
- Legend panel shows item name, icon, size, and percentage
- Cross-highlighting: hovering a slice highlights its legend row and vice versa
- Chart auto-rebuilds when folder size calculations complete
- Works in single-pane and split-pane modes
- Pure WPF implementation — no third-party charting library

DIAGNOSTICS
- Options > Export Diagnostics bundles logs and config into a zip on the Desktop
- Zip includes: operational.log, crash_dump.log, settings.json, backup_log.json, system_info.txt
- system_info.txt records: app version, OS, .NET version, machine name, uptime, RAM, CPU count,
  monitor list, and drive inventory
- Zip written to a temp file first and moved atomically — no partial zip left on failure
- Unclean-shutdown sentinel: MarkRunning() writes running.lock on startup;
  MarkExitedCleanly() removes it on clean exit; ConsumeUncleanShutdown() detects crash-restarts

APPLICATION
- Dark and Light theme support with runtime toggle
- Theme preference persisted to settings
- Light mode polish: warmer grays, dedicated brush keys for toggles, drive usage rings, and donut chart center
- System tray support — close/minimize hides to tray; tray icon right-click menu shows Show/Hide and Quit; only exit via Quit in tray menu
- Single-instance enforcement — second instance signals first instance to show its window and foreground, then exits; no "already running" error
- Smart startup behavior — --startup flag (set by Launch at Startup shortcut) hides to tray on login; manual launch always shows window; missed backups override tray-hide to show window
- GitHub Releases update checker — checks on startup; accent-colored banner with Download, Dismiss, Skip Version options; skipped versions persisted; also accessible via Options > Check for Updates
- Crash logging to %LocalAppData%\Beet's Backup\crash_dump.log
- Operational logging to %LocalAppData%\Beet's Backup\operational.log (10MB rotation)
- FileLogger.LogDirectory (public) used by "Open Log Folder" feature
- Atomic JSON saves (settings, jobs, log) — crash-safe
- Portable single-file self-contained deployment (win-x64)
- ServiceProvider disposed on app exit (all singleton services cleaned up properly)
- Custom window with Segoe UI Variable font
- Assembly version 3.0.0
- Namespace disambiguation — explicit aliases prevent WinForms/WPF type collisions
- 132-test xUnit suite (BeetsBackup.Tests/)

================================================================================
SECTION 2: KNOWN LIMITATIONS
================================================================================

- VSS requires elevation — if the app is not running as Administrator, the locked-file VSS
  fallback is unavailable; locked files will be detected and reported but not backed up
- Compression creates a single .zip archive — individual file timestamps and attributes are
  not preserved inside the zip
- Windows Task Scheduler integration fires even without the app running, but the app must be
  installed at a stable path (task registration uses the current exe path)
- No cloud backup support (local/USB/network drives only)
- No bidirectional sync (one-way copy/move only)
- No incremental/differential backup (always full comparison)
- Cannot open files from within the app (files must be opened via Windows Explorer)
- No file preview/thumbnail support
- Update checker notifies but does not auto-install updates (user must download manually)
- Recycle Bin delete called from thread pool — COM dialog could fail on non-STA thread (rare edge case)
- BackupLogEntry.Id is get-only — deserialized entries get new GUIDs (doesn't affect current functionality)
- No file filtering by date range or size range
- No backup verification/restore workflow beyond per-file versioning
- Scheduler requires app to be running for in-process scheduling; Windows Task Scheduler integration
  handles the "app closed" case but is not a full Windows Service
- Windows only (WPF, .NET 8)

SECTION 3: CRITICAL TESTING AREAS
================================================================================

MOVE OPERATION (highest risk — data loss)
- Move operation: Verify source is NOT deleted when files fail to copy (test with read-only dest, locked files, disk full simulation)
- Move operation: Verify source IS deleted when all files copy successfully

ATOMIC SAVES
- Atomic saves: Kill the app mid-save and verify JSON files are not corrupted on next launch

SCHEDULER
- Scheduler: Run multiple scheduled jobs simultaneously, verify no crashes or data corruption
- Scheduler: Verify scheduler recovers after an exception (doesn't die silently)
- Scheduler: Verify SchedulerError event fires and message appears in status bar on job failure
- Recurring job: Set a 6-hour recurring job, verify it fires on schedule and reschedules
- Job snapshot: Mutate a scheduled job's settings mid-run, verify the running job is unaffected

SINGLE INSTANCE
- Single-instance: Launch two copies, verify second shows message and exits

TRANSFER MODES
- Skip Existing: Test with same-size files that have different timestamps — should copy
- Skip Existing: Test with same-size, same-timestamp files — should skip
- Skip Existing: After a copy, run again immediately — verify no files are re-copied (timestamp preservation)
- Keep Both: Verify files get -1, -2 suffixes correctly
- Replace: Verify existing files are overwritten
- Mirror: Verify destination-only files are deleted after confirmation
- Mirror: Verify source files not in destination are copied

DELETE / RECYCLE BIN
- Recycle Bin: Verify deleted files appear in Windows Recycle Bin, not permanently deleted

SEARCH
- Deep search: Search a large directory (10,000+ files) and verify UI stays responsive
- Deep search: Cancel mid-search and verify clean cancellation

FOLDER TREE
- Folder tree: Expand a folder on a slow USB drive, verify UI doesn't freeze
- Folder tree: Expand a folder on a network drive, verify no hang
- Drive list: Eject a drive mid-enumeration during Refresh, verify remaining drives still load

TRANSFER PROGRESS
- Transfer progress: Verify progress bar reaches 100% on completion
- Transfer progress: Verify ETA is reasonable
- Pause/Resume: Pause mid-transfer, resume, verify completion is correct
- Pause/Resume: Start a second transfer after one completes, verify no ObjectDisposedException
- Stop: Cancel mid-transfer, verify partial files are handled cleanly

ERROR HANDLING
- Disk full: Fill destination drive mid-transfer, verify error reported in log and "View Errors" shows the file
- Locked files: Copy a folder containing a file locked by another process, verify it's reported in "View Errors"
- Checksum mismatch: Simulate a checksum failure, verify it appears in "View Errors"
- Per-file errors: Trigger 200+ file errors, verify the list is capped at 200
- UNC path: Schedule a backup to a \\server\share destination, verify no crash in free space check

COPY OPTIONS
- Checksum verify: Copy files with verify enabled, confirm SHA256 matches
- Permission stripping: Copy files with permissions option, verify ACLs are reset
- Hidden files: Copy hidden files/folders, verify hidden attribute preserved at destination
- Timestamp preservation: Copy a file; verify destination LastWriteTime matches source

BANDWIDTH THROTTLING
- Throttle: Enable 1 MB/s limit, copy a large file, verify transfer rate stays near the cap
- Throttle: Set per-job speed in scheduler, verify rate respected during scheduled run

BACKUP WIZARD
- Open wizard, choose "One-time now" — verify Schedule step is skipped; finish button says "Start Backup Now"
- Open wizard, choose "Recurring" — verify Schedule step appears; finish button says "Schedule Backup"
- Leave Source empty and click Next — verify validation blocks progress with no crash
- Enter a destination on the same drive as the source — verify same-drive warning appears
- Reach Review step — verify summary populates correctly and size estimate runs asynchronously
- Complete wizard with a destination that has insufficient free space — verify warning dialog appears;
  user can cancel (no job created) or proceed (job created despite warning)
- Complete wizard with "One-time now" — verify job runs immediately and does not appear in Scheduled Jobs list
- Complete wizard with "Recurring" — verify job appears in Scheduled Jobs list with correct next-run time

FILE VERSIONING
- Enable versioning on a job, run it twice — verify .versions folder appears at destination and
  contains one archived copy of the first run's output
- Verify .versions folder has the Windows Hidden attribute set
- Run with MaxVersions=2, trigger 4 transfers to the same destination — verify only 2 archived
