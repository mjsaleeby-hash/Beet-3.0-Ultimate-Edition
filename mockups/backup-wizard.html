What kind of backup do you want to set up?

Don't worry — you can change this later. Pick the option that sounds most like what you need.

⚡
Back up right now, just once
Copy your files to a safe place immediately. Great for before you install new software, upgrade Windows, or take your laptop somewhere.
📅
Back up on a specific date and time
Schedule it once — for example, tonight at midnight, or next Sunday at 10 PM — and the app will run it automatically while you sleep.
🔁
Back up automatically, on a schedule Recommended
The app will back up your files every day, every week, or on whatever schedule you choose. Set it and forget it — the safest option for keeping your files protected.
💡 Not sure? Choose "automatically on a schedule" — daily backups mean even if something goes wrong today, yesterday's copy is always safe.

What should happen if a file already exists in the backup?

This controls what Beet's Backup does when it finds a file at the destination that has the same name as one it's about to copy.

⏭️
Skip files that are already there Recommended
If a file named "photo.jpg" already exists in the backup, leave it alone and move on. Only copies files that are brand new.
Best for: most situations. Fast, safe, never overwrites anything.
📋
Keep both copies
If "photo.jpg" already exists, the new copy is saved as "photo (2).jpg". Both versions are kept.
Best for: one-time backups where you want to keep all versions. Avoid for recurring — the destination grows larger each run.
🔄
Replace with the newer version
If the file in the backup is older than the source file, overwrite it with the updated version.
Best for: keeping the backup fully up-to-date. The destination always has the latest copy of every file.
🪞
Mirror — make the destination identical
After copying, delete anything in the destination that doesn't exist in the source. The destination becomes an exact clone of the source.
Best for: keeping a perfect sync copy. Use with care — files you place in the backup folder manually will be deleted.
🗑️ Mirror will permanently delete files from the destination that are not in the source — including files you placed there manually. This action cannot be undone. Are you sure this is what you want?

Optional: fine-tune your backup

These settings are all turned off by default. Most people don't need to change anything here. Skip this step if you're not sure.

✓
Verify file integrity after copying
After every file is copied, Beet's Backup re-reads both the original and the copy and checks they are byte-for-byte identical (using a SHA-256 checksum). If they don't match, the backup is flagged as failed.

When to enable: If you're backing up to an older or unreliable drive, or if these files are critical and you need absolute certainty they copied correctly.
Trade-off: Makes the backup roughly twice as slow, because every file is read twice.
✓
Remove access restrictions on copied files
Windows files can be "locked" to a specific user account — meaning only you (on this specific PC) can open them. This option removes that restriction so the backed-up files can be opened by anyone on any computer.

When to enable: If you plan to restore files on a different computer, or share the backup with someone else. Otherwise, leave this off.
✓
Limit transfer speed
Slows down the backup so your computer stays responsive while it runs. Useful if the backup happens while you're working or gaming and you don't want it to eat up all your disk bandwidth.
Max speed: (Full speed if unchecked)
✓
Skip certain types of files
Tell Beet's Backup to ignore files matching a pattern — so they won't be copied to the destination. Useful for skipping temporary files that waste space.

Examples: *.tmp Thumbs.db node_modules *.log
