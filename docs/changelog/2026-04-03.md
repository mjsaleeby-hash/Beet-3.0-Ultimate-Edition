The historical record below is preserved as-is.

## New Features

### VSS Shadow Copy Fallback for Locked Files

- Locked or in-use files (e.g. open Outlook PST files, live database files) are now handled automatically during transfers
- **Retry logic:** each locked file gets 3 copy attempts with 500 ms delays between attempts before the fallback triggers
- **VSS fallback:** if retries are exhausted, a Volume Shadow Copy snapshot of the source volume is created so the file can be read without interrupting the owning process
- **Per-volume snapshot caching:** snapshots are created once per volume per transfer session and reused for all subsequent locked files on that volume; no redundant snapshot overhead
- **Automatic cleanup:** all VSS snapshots created during a session are deleted after the transfer completes, regardless of success or failure
- **Transfer summary reporting:** the completion summary now shows "X via shadow copy" so users know how many files required the fallback
- **No external packages:** implemented via P/Invoke to `vssapi.dll`, which ships with Windows; zero additional dependencies

### Simple / Advanced Toolbar Mode

- A persistent **Simple/Advanced toggle switch** is now present on the toolbar
- **Simple mode** shows five buttons: Visual, Theme, Split Pane, Refresh, and Backup Wizard — a clean, minimal interface for everyday use
- **Advanced mode** shows the full toolbar: schedule, jobs, log, options, pause/stop controls, and the permissions/checksums/throttle toggles
- The selected mode is **persisted in settings** and restored across app restarts
- `SettingsService` carries the new `ToolbarMode` preference

### Backup Wizard Button (Placeholder)

- A **Backup Wizard** button appears in Simple toolbar mode only
- Styled with an electric purple "blacklight" badge, matching the SOURCE/DESTINATION pane label design language
- Currently a placeholder for a future guided backup setup flow; clicking it will surface the wizard when implemented
