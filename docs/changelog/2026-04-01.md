# Changelog — 2026-04-01

## Bug Fixes

### Critical

#### Race Condition in SchedulerService (Job Snapshot)
- Introduced `SnapshotJob()` which clones job data (source, destination, transfer mode, filters, etc.) into a plain struct before handing it to a background task
- `ExecuteJobAsync` reads exclusively from the snapshot; only `LastRun` is written back to the original object, under lock
- Eliminates concurrent read/write on shared `ScheduledJob` instances between the UI thread and the timer thread

#### `_pauseGate` ObjectDisposedException in MainViewModel
- `BeginTransfer()` now creates a fresh `ManualResetEventSlim` for every transfer, disposing the previous instance after signaling it open
- Previously the same gate instance was reused across transfers; a second transfer starting after the first completed could try to wait on a disposed event

#### SaveJobs Serialization Race
- Resolved as a side effect of the job snapshot fix above: background tasks no longer hold references to shared job objects while `SaveJobs` is serializing them

---

### High

#### Infinite Loop on `TimeSpan.Zero` in UpdateNextRun
- Added a `> TimeSpan.Zero` guard in `UpdateNextRun` so a job with a zero interval cannot spin the scheduler loop indefinitely

#### BackupLogService UI-Thread I/O Jank
- `Save()` now debounces with a 1-second delay before writing to disk, using a deferred dispatcher callback
- Rapid status updates during an active transfer no longer trigger a file write on every tick

#### PropertyChanged from Background Thread in FileSystemItem.Size
- `CalculateDirectorySizeAsync` now marshals the `Size` setter onto the UI thread via `Dispatcher.InvokeAsync` before raising `PropertyChanged`
- Prevents cross-thread binding exceptions when folder size calculations complete on a thread pool thread

#### CheckFreeSpace Crashes on UNC / Network Paths
- `DriveInfo` constructor in `TransferService.CheckFreeSpace` is now wrapped in a `try/catch` for `ArgumentException`
- Free space check is silently skipped for UNC paths and other paths that `DriveInfo` cannot resolve

---

### High (continued)

#### Scheduler Reads/Mutates NextRun and IsEnabled Before Task Reads Original Values
- Covered by the job snapshot approach described above

---

### Medium

#### Null `Application.Current` During Shutdown
- `CalculateFolderSizesWithProgressAsync` now null-checks `Application.Current` before dispatching, preventing a `NullReferenceException` when the app is shutting down while a size calculation is still in flight

#### SchedulerService Never Disposed
- `App.OnExit` now calls `Dispose()` on the `ServiceProvider`, which triggers `IDisposable.Dispose()` on all registered singleton services including `SchedulerService`
- The scheduler's `CancellationTokenSource` and timer are now properly torn down on app exit

#### TimeSpan Locale Sensitivity in NullableTimeSpanConverter
- `NullableTimeSpanConverter` now uses `CultureInfo.InvariantCulture` for both read and write
- Prevents scheduled job intervals from being misread on machines with non-English locale settings (e.g., comma as decimal separator)

#### CopyFile Timestamp Preservation
- `FileSystemService.CopyFile` now calls `File.SetLastWriteTimeUtc` after `File.Copy`
- Previously, the destination file's last-write time was set to the copy time; `Skip Existing` mode comparing timestamps would re-copy every file on the next run

#### CancellationTokenSource Leak
- Old `CancellationTokenSource` instances in navigation and search paths are now disposed before new ones are created

---

### Low

#### Drive Ejection Race in LoadDrives
- `LoadDrives` now wraps each individual drive enumeration in a `try/catch` for `IOException`
- A drive ejected mid-enumeration no longer prevents the rest of the drive list from loading

#### CSV Newline Escaping in Log Export
- The `Escape()` helper in `LogDialog.xaml.cs` now strips `\r` and replaces `\n` with a space
- Previously, multi-line strings in log entry fields would break CSV row structure

## New Features

### 1. "Open Log Folder" Button (Log Dialog)
- New button added to the Log dialog, positioned between the action buttons and the Export CSV button
- Opens `%LocalAppData%\Beet's Backup\` directly in Windows Explorer
- Gives one-click access to `operational.log`, `crash_dump.log`, `scheduled_jobs.json`, and `backup_log.json`
- `FileLogger.LogDir` renamed to `FileLogger.LogDirectory` (now `public`) to support this feature

### 2. Scheduler Error Surfacing (Status Bar)
- `SchedulerService` now exposes a `SchedulerError` event that fires when:
  - The scheduler loop itself encounters an unhandled exception
  - A scheduled job fails (both `InsufficientSpaceException` and general exceptions)
- `MainViewModel` subscribes to `SchedulerError` and displays the message in the status bar, marshaled to the UI thread
- Subscription is properly cleaned up in `MainViewModel.Dispose()`

### 3. Per-File Error Tracking
- `TransferResult` now carries a `List<FileError>` capped at 200 entries, populated via `AddFileError(path, reason)`
- `FileError` is a new class with `Path` (string) and `Reason` (string) properties
- `TransferService.CopyItem` records a `FileError` at every catch point: disk full, locked file, general I/O failure, and checksum mismatch
- `BackupLogEntry` now has a `FilesFailed` count and a `List<FileError> FileErrors` property
- `StatsDisplay` shows "Complete with N error(s)" when failures are present (previously only showed "Complete")
- `BackupLogService.UpdateStats` accepts optional `filesFailed` and `fileErrors` parameters
- New red **"View Errors"** button in the Log dialog — enabled only when the selected log entry has file errors; clicking it shows a `MessageBox` listing each failed file's path and reason
