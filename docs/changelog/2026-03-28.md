## Bug Fixes

### Fix: Backup Progress Showing 99%

- Fixed integer division truncation in progress percentage calculation that caused 99% to display instead of 100%
- Fixed missing `Message` update in `UpdateStats` so the status message stays current throughout the transfer
- Added a migration pass on log load to correct existing persisted log entries that stored the wrong progress value

---

## New Features

### Missed Backup Detection

- On app startup, detects any scheduled jobs whose scheduled run time was missed while the app was closed
- Shows a dialog listing all missed jobs with **"Run All Now"** and **"Skip All"** options
- Tracks `LastRun` per job so detection is accurate across app restarts
- Works for both recurring and one-time jobs

### Mirror (Sync) Transfer Mode

- New `TransferMode.Mirror` that copies new and changed files to the destination, then **deletes any destination files not present in the source**
- Before starting, shows a confirmation dialog with a prominent bordered warning callout making clear that destination-only files â€” including manually-placed files â€” will be permanently deleted
- Available in both the manual transfer dialog and the schedule dialog

### Exclusion Filters

- Skip files during backup by **extension pattern** (e.g. `*.tmp`, `*.log`) or **exact name** (e.g. `Thumbs.db`, `node_modules`)
- Configurable per scheduled job in the schedule dialog
- Filters are applied during both the file-counting pass and the actual copy pass
- UI includes Enter-key support for adding filters and example hint text

### Backup Verification in Scheduled Jobs

- SHA-256 checksum verification (already present in the engine for manual transfers) is now exposed as a **checkbox in the schedule dialog**
- Includes a disclosure note about the performance cost
- Setting is passed through `SchedulerService` to `TransferService` at run time

### Backup Size Estimation

- **"Estimate Size" button** in the schedule dialog calculates the total byte size and file count for the configured source, respecting any active exclusion filters
- Auto-estimation also runs when a scheduled job starts; the result is included in the opening log message

### Pause / Resume for Scheduled Backups

- Scheduled backups now receive a `ManualResetEventSlim` pause gate so they can be paused and resumed mid-run
- **Pause button** added to the log dialog; always visible, enabled only when a currently running job is selected
- Reuses the existing engine-level pause/resume support already present for manual transfers
