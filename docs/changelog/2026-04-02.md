# Changelog — 2026-04-02

## Bug Fixes

### Pie Chart — Duplicate Legend Entries on Rapid Navigation

- `BuildPieSlices` now constructs a new `ObservableCollection<PieSlice>` and assigns it atomically to `TopPieSlices` / `BottomPieSlices` instead of calling `Clear()` followed by individual `Add()` calls
- The previous approach triggered a `CollectionChanged` notification per item; rapid folder navigation could interleave two rebuild passes, causing legend rows to appear twice
- The atomic swap means the binding sees exactly one collection-replacement notification per rebuild

### Ghost Process — Disposal Deadlock on App Exit

- `App.OnExit` now runs `ServiceProvider.Dispose()` on a background thread with a 5-second timeout rather than calling it directly on the UI thread
- The old path blocked the UI thread while waiting for `SchedulerService.Dispose()` to join its timer loop; if a scheduled job was mid-execution the timer task was itself waiting on the UI thread, creating a deadlock that left the process alive after the window closed ("ghost process")
- A warning is logged if disposal does not complete within the timeout

## New Features

### Search — Path Column in Results

- A hidden **Path** column (`DirectoryPath` binding) is now present in all three file list `GridView` instances (`TopPathColumn`, `SplitTopPathColumn`, `SplitBottomPathColumn`)
- The column has `Width="0"` during normal folder browsing and is made visible when a deep search is active, showing the parent folder of each result so users can tell where matching files live
- Uses `FileSystemItem.DirectoryPath`, a computed property that returns `Path.GetDirectoryName(FullPath)`

### Search — "Open in Explorer" Context Menu Option

- Right-click context menu now includes **"Open in Explorer"**
- For folders: opens the folder directly in Windows Explorer
- For files: launches Explorer with the file pre-selected (`explorer.exe /select,"<path>"`)
- Available in all pane contexts (single-pane top list, split-pane top list, split-pane bottom list)

### Search UX Improvements

- `ExecuteDeepSearchCommand` is decorated with `AllowConcurrentExecutions = true` so the command does not block while a search is in progress; a new search cancels and replaces the previous one immediately
- Searching without a folder loaded now shows the status message `"Select a drive or navigate to a folder first, then search."` rather than silently doing nothing
- Status bar reports live progress: `"Searching \"<path>\" for \"<query>\"..."` during the search and `"Found N result(s) for \"<query>\""` or `"No results for \"<query>\""` on completion

### Launch at Startup

- New **"Launch at Startup"** toggle in the Options dropdown menu (next to the theme toggle in the main toolbar)
- When enabled, `SettingsService` creates a Windows shortcut (`Beet's Backup.lnk`) in the user's startup folder (`%AppData%\Microsoft\Windows\Start Menu\Programs\Startup`)
- When disabled, the shortcut is deleted
- On launch, if `LaunchAtStartup` is set and there are no missed backups to handle, the window opens **minimized** so the app stays out of the way
- If missed backups are detected on a startup launch, the window opens at normal size so the missed-backup dialog is visible
